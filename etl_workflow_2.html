Cloud-Based Financial Transactions Data Pipeline

Project Goal: Build a robust, scalable, and automated data pipeline to ingest, validate, and transform multi-currency financial transactions with daily reporting and cost-efficient processing.

Solution: Used AWS S3 for storing raw and processed data, EC2 for lightweight validation, EMR with PySpark for transformation and currency normalization using daily exchange rates, and Apache Airflow to orchestrate EC2/EMR automation, dependency setup, and job execution. Data was partitioned and stored in Parquet for optimized query performance via Athena.

Impact: Automated end-to-end processing of daily transactions, improved data quality through validation layers, reduced EMR compute costs with incremental loading and partition pruning, and enabled accurate, on-time reporting to support business decisions.

Key Skills & Deliverables:

  • AWS EC2, EMR, and S3
  • Apache Airflow for orchestration
  • PySpark for transformation and aggregation
  • Pandas for data validation
  • Partitioned Parquet and Athena for efficient querying
  • Boto3 and REST API for exchange rate integration
